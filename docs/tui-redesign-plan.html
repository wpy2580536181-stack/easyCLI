easyCLI TUI 改进方案

对标 OpenCode · QoderCLI · Claude Code 的交互模式与设计理念 ｜ 结构化的视觉与交互设计规格
基线：Ink 5.2.1 + @inkjs/ui 2.0.0 配色：TOKENS（青 #22D3EE / 洋红 #EC4FD0） 组件链：SplashTitle→Transcript→StatusLine→StatusBar→InputBox→Approval 2026-07-14

0 · 现状基线与设计目标

easyCLI 当前的 TUI 已经具备清晰的组件分层、应用内滚动（PageUp/Down·Home/End）、斜杠命令下拉、历史翻页、流式 Markdown 渲染与 Approval 确认弹窗。本方案在不破坏现有 470+ 测试与裁剪数学的前提下，补齐三类成熟 CLI 已经验证、而当前缺失的体验维度：会话管理、代码差异可视化、文件/git 侧栏，并把输入交互、进度反馈、主题与布局自适应统一到一套设计令牌上。

  • 现状能力：状态栏（模型·分支·ctx%·token·时长·模式）、流式输出、工具调用 Badge、滚动回看、/命令下拉、Ctrl+C 中断。
  • 主要缺口：无历史会话切换 / 对话树；无 diff 视图；无文件树与 git 状态；多行输入弱、无 @ 引用与历史搜索；长任务无进度条/取消；布局固定无侧栏自适应。

01会话管理（Session Management）

参考：Claude Code 的对话树（branch / checkpoint，可回退另开分支）· OpenCode 的左侧会话面板（列表切换 + 新建）。

设计要点

  • 会话面板：侧栏下半区列出历史会话（标题 / 最近时间 / token 占用），↑↓ 选择，Enter 切换。
  • 对话树：主区按轮次展示；分叉点用 ⌥ 标记「可回退 / 另开分支」。
  • 折叠 / 展开：每个用户轮次可折叠其下助手回复；Alt+↑/↓ 或 [ ] 折叠，长上下文自动 compact。
  • 持久化：会话存 .easycli/sessions/*.jsonl，恢复时重载 transcript。

键位与标记

操作快捷键
打开会话面板Ctrl+O
新建会话Ctrl+N
搜索会话Ctrl+K
折叠 / 展开轮次Alt+↑/↓

当前会话：primary 反显；历史会话：subtext；分支点：brand 图标 ⌥。

02代码差异展示（Diff Visualization）

参考：QoderCLI 行内 diff 高亮 · Claude Code 并排对比视图。

可视化方案

  • 默认行内（unified）diff：左侧 gutter [+]/[-]/ ；新增行左侧绿条 + 浅绿底，删除行左侧红条 + 浅红底，修改行用 ~ + 行内变更字符高亮。
  • 并排（split）视图：左右两栏、中间分隔、行号对齐；Ctrl+D 或 /diff split 切换。
  • 行号列：旧 / 新行号（subtext）。
  • 语法高亮：diff 内代码沿用 Markdown 着色，保证可读性。
  • 折叠与截断：@@ … @@ hunk header 可点击折叠大段上下文；超 N 行截断 + 「↓ 展开剩余 M 行」。

03文件操作面板（File Panel）

参考：OpenCode 侧边文件树 · QoderCLI 把 git 状态直接集成进文件行。

设计要点

  • 文件树：目录折叠/展开，当前打开文件 primary 反显。
  • git 状态：文件名左侧角标 M/A/D/R/?/U，用语义色。
  • 暂存区指示：已 stage 文件加 ✓；/stage /unstage 或 Space 切换。
  • 快速跳转：Ctrl+P fuzzy find file，复用输入下拉机制。
  • 统计条：栏顶「12 modified · 3 staged」。
侧栏文件树 + git 状态

git 状态配色：M 黄、A 绿、D 红、R 蓝、? 灰、U 橙；树连线 subtext；选中行 primary 反显。窄终端（<100 列）默认隐藏侧栏，Ctrl+B 手动切换。

04输入交互体验（Input UX）

参考：Claude Code 输入框（多行、@、/、历史搜索）。

机制设计

  • 多行输入：Enter 提交；Shift+Enter / Alt+Enter 换行；续行用 ║ 标记（借鉴 Claude Code）。
  • 命令补全：/ 触发下拉，按分组（会话/文件/模型/工具）。
  • 历史搜索：Ctrl+R 进入 fuzzy 搜索，复用 InputBox 下拉，Enter 填入。
  • @ 引用：输入 @ 触发路径补全，选中渲染为 chip（青底反显），提交时展开。
  • 扩展命令：/file /stage /session 等。
多行 + @chip + /下拉

05进度与状态反馈（Progress & Status）

参考：三者均用 spinner + 流式 + 进度条；Claude Code 的取消提示最明确。

  • 工具调用状态：StatusLine 已有 spinner + 模式 Badge；增强——并行工具用多 chip，工具名 value 白、参数折叠。
  • 流式输出：逐 token 渲染 + 行尾闪烁 caret（已具备 Markdown 实时渲染）。
  • 长任务进度条：多文件编辑 / 索引用 [####----] 42% 文本进度条，Ctrl+C 可中断。
  • 取消机制：进行中显示「Esc 取消 / Ctrl+C 中断」；Approval 已有确认弹窗。
  • 完成反馈：工具完成打 ✓(success)；失败 ✗(error) + 折叠错误详情。

  • 语义层级：primary > brand > status > text > subtext，避免「处处高亮」无重点。
  • 图标：统一用 Nerd Font / 文本符号（● ◆ ▸ ✔ ✗ ⌥ ║ ⠿），避免 emoji 依赖以保证终端兼容。
  • 边框与间距：与 splash 一致方角 ┌─┐│└┘；区域间 1px 灰分割线；padding 1。
  • 焦点高亮：选中项 = inverse 反显或 primary 左条；当前面板标题 primary bold。
  • 明暗预留：当前深底 #161320；TOKENS 抽象为语义，预留 light 映射。
  • 一致性铁律：所有组件引用 TOKENS / theme.tsx palette，禁止散落裸色名（如 color="cyan"）。

07布局结构与尺寸自适应（Layout）

区域划分

  • 顶部状态栏（Top Bar）：模型·分支·ctx%·token·时长·模式（StatusBar 增强）。
  • 侧边栏（Sidebar，可选）：文件树 + git 状态 + 会话切换（Ctrl+B）。
  • 主内容区（Main）：Transcript（对话树/折叠）+ Diff 视图 + 工具调用卡，flexGrow。
  • 底部状态行（Footer）：StatusLine（spinner + 状态 + 取消提示）。
  • 输入区（Input）：多行输入 + 下拉（slash / @ / history），固定底部。

尺寸自适应策略

终端宽度布局决策
≥ 120 列侧栏 24 列 + 主区；状态栏单行全字段。
80 – 119 列侧栏 20 列或默认隐藏；状态栏压缩（省略时长）。
< 80 列隐藏侧栏，状态栏仅模型 + 模式；输入单行优先。
高度 < 24 行状态栏合并进 footer，避免双层占用。

布局用 Ink flex：主区 flexGrow、侧栏固定宽、输入区固定底部；所有区域宽度由 store.width 推导，沿用现有 buildVisibleLines 裁剪数学。
