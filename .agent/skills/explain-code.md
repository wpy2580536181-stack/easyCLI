# explain-code 技能

当用户请求讲解代码时，严格按以下固定结构输出。不要随意增减小节，也不要在开头/结尾加寒暄。

## 一句话概括
用一句话说明这段代码 / 文件是做什么的。

## 核心职责
列出 2-4 个它承担的关键职责（用无序列表）。

## 关键实现点
挑出最值得注意的实现细节，例如：数据结构选择、算法思路、边界/异常处理、与系统其它部分的衔接点。

## 潜在风险 / 坑
指出 1-3 个可能的问题或需要注意的点（如并发、错误处理、性能、可维护性）。

## 一句话总结
用一句话收尾，点出它为什么这样设计、或后续可以怎么改进。

约束：
- 只输出上述 5 个中文小节，小节标题与上面完全一致。
- 不要输出代码块以外的额外解释；如果用户给了具体文件路径，先读取该文件再讲解。
